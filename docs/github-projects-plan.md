# GitHub Projects Plan

## Recommendation
Use the newer GitHub Projects experience rather than classic projects.

## Why
- better long-term flexibility
- custom fields and better filtering
- more natural fit for a Mission Control style dashboard
- stronger support for future visual work tracking

## Initial board setup

### Project name
Mission Control Sprint Board

### Suggested fields
- Status
- Priority
- Area
- Role
- Sprint

### Suggested status options
- Inbox
- Ready
- In Progress
- Review
- Blocked
- Done

### Suggested area options
- Frontend
- Backend
- Design
- Product
- Ops
- Docs

### Suggested role options
- Chief of Staff
- Product Lead
- Design and UX
- Frontend Engineering
- Backend Engineering
- QA and Review
- DevOps and Automation

## Recommended views
- Board view by Status
- Table view for planning
- Filtered view for high priority work
- Filtered view for blocked work

## Immediate next setup steps
1. Create the GitHub Project.
2. Add the core fields.
3. Seed initial issues from `mission-control-issue-seed.md`.
4. Place Sprint 1 issues into the board.
5. Add direct board and issue links into Mission Control UI once stable URLs are available.
