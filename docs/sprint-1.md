# Mission Control Sprint 1

## Sprint goal
Ship a polished static Mission Control shell that clearly visualizes active work, team structure, and project status.

## Priority outcomes
- reusable dashboard shell
- overview page with meaningful status sections
- dedicated Projects view
- visible GitHub and Discord link surfaces
- strong visual identity aligned with the dark operational direction

## Sprint backlog

### In progress
- Componentize the overview page into reusable layout and dashboard pieces
- Add a dedicated Projects screen
- Wire real GitHub and Discord links into the UI where available

### Ready next
- Add Tasks screen shell
- Add Team screen shell
- Replace placeholder links with deeper issue and board links once available
- Seed GitHub issues from `mission-control-issue-seed.md`

### Blocked or pending decisions
- Finalize GitHub Projects configuration
- Decide when to add real GitHub issue and board data ingestion

## Definition of success
Sprint 1 is successful when:
- the app feels like a real control surface
- project work is easy to visualize
- the structure is reusable enough to support additional screens quickly
- GitHub and Discord are visibly part of the product workflow
